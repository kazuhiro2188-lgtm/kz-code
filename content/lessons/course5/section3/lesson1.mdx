# 可用性の概念とSLA

## はじめに

このレッスンでは、**可用性**の概念と**SLA（Service Level Agreement）**について学びます。

## 可用性とは

**可用性**は、システムが利用可能である時間の割合です。

### 可用性の計算

```
可用性 = (総時間 - ダウンタイム) / 総時間 × 100%
```

### 可用性のレベル

- **99%**: 年間約87.6時間のダウンタイム
- **99.9%**: 年間約8.76時間のダウンタイム
- **99.99%**: 年間約52.56分のダウンタイム
- **99.999%**: 年間約5.26分のダウンタイム

## SLAとは

**SLA**は、サービスプロバイダーと顧客の間で合意されたサービスレベルの合意です。

### SLAの要素

- **可用性**: システムが利用可能である時間の割合
- **パフォーマンス**: レスポンスタイム、スループット
- **サポート**: サポートの応答時間、解決時間

### SLAの例

```
可用性: 99.9%
レスポンスタイム: 200ms以下（95パーセンタイル）
サポート応答時間: 1時間以内
```

## 可用性を向上させる方法

### 1. 冗長化

複数のリソースを用意し、1つが故障しても他のリソースで動作を継続します。

### 2. フェイルオーバー

故障したリソースから正常なリソースに自動的に切り替えます。

### 3. ヘルスチェック

リソースの状態を定期的に確認し、異常を検出します。

### 4. バックアップと復旧

データのバックアップを取得し、障害時に復旧します。

## AI開発における可用性

### 推論サービスの可用性

- **複数の推論サーバー**: 1つのサーバーが故障しても他のサーバーで動作継続
- **ロードバランサー**: リクエストを複数のサーバーに分散
- **ヘルスチェック**: 推論サーバーの状態を監視

### 学習サービスの可用性

- **チェックポイント**: 学習途中の状態を保存
- **分散学習**: 複数のサーバーで分散学習し、1つが故障しても継続
- **データのバックアップ**: 学習データのバックアップ

## まとめ

- **可用性**はシステムが利用可能である時間の割合
- **SLA**はサービスレベルの合意
- 冗長化、フェイルオーバー、ヘルスチェックにより可用性が向上
