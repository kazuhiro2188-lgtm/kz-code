# 効果的なプロンプト設計

前回学んだプロンプトの基本を踏まえ、より効果的なプロンプトを設計する方法を学びます。

## プロンプトの構造化

効果的なプロンプトは、以下の要素を含むことが推奨されます：

1. **役割の定義**: AIにどのような役割を担ってもらうかを明確にする
2. **コンテキストの提供**: 必要な背景情報を提供する
3. **タスクの明確化**: 何をすべきかを具体的に示す
4. **出力形式の指定**: 期待する出力の形式を指定する

## 実践例

### 例1: コードレビュー

```
あなたは経験豊富なソフトウェアエンジニアです。
以下のTypeScriptコードをレビューし、改善点を指摘してください。

コード:
[コードをここに貼り付け]

出力形式:
- セキュリティ上の懸念
- パフォーマンスの問題
- コードの可読性
- 改善提案

各項目について、具体的な例とともに説明してください。
```

### 例2: ドキュメント作成

```
あなたは技術文書作成の専門家です。
以下のAPI仕様に基づいて、開発者向けのドキュメントを作成してください。

API仕様:
[API仕様をここに記載]

出力形式:
- 概要
- エンドポイント一覧
- リクエスト/レスポンスの例
- エラーハンドリング
- 使用例

Markdown形式で出力してください。
```

## 高度なテクニック

### 1. 段階的な指示

複雑なタスクは、小さなステップに分割して指示します。

```
ステップ1: 要件を分析してください
ステップ2: 設計案を3つ提示してください
ステップ3: 各設計案のメリット・デメリットを比較してください
ステップ4: 最適な設計案を選択し、実装コードを提示してください
```

### 2. 例示の活用

期待する出力の例を示すことで、AIの理解を深めます。

```
以下の形式でJSONを出力してください：

例:
{
  "status": "success",
  "data": {
    "id": 123,
    "name": "サンプル"
  }
}
```

## まとめ

効果的なプロンプト設計により、AIとの協働をより効率的に行うことができます。次回は、AIツールの選び方について学びます。
