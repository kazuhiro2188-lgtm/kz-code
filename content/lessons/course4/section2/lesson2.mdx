# ベクトルDBの基礎

## はじめに

このレッスンでは、**ベクトルDB（Vector Database）**の基礎について学びます。

## ベクトルDBとは

**ベクトルDB**は、高次元のベクトルデータを効率的に格納・検索するためのデータベースです。

### 主な特徴

- **ベクトル検索**: 類似度検索が高速
- **高次元対応**: 数百から数千次元のベクトルを扱える
- **近似最近傍検索**: 高速な類似度検索

## ベクトルとは

**ベクトル**は、数値の配列で表現されるデータです。

**例:**
```
[0.1, 0.5, 0.3, 0.9, 0.2, ...]
```

AIモデル（特に埋め込みモデル）は、テキストや画像をベクトルに変換します。

## ベクトル検索

ベクトルDBは、ベクトル間の類似度を計算して検索します。

### 類似度の計算方法

- **コサイン類似度**: ベクトルの角度で類似度を測定
- **ユークリッド距離**: ベクトル間の距離で類似度を測定
- **内積**: ベクトルの内積で類似度を測定

## 主要なベクトルDB

- **Pinecone**: マネージドサービス、高パフォーマンス
- **Weaviate**: オープンソース、グラフ機能も持つ
- **Qdrant**: オープンソース、Rust実装
- **Milvus**: オープンソース、スケーラブル

## 使用例

### セマンティック検索

テキストの意味に基づいて検索します。

```typescript
// クエリをベクトルに変換
const queryVector = await embedText("AI開発について");

// 類似するベクトルを検索
const results = await vectorDB.search(queryVector, { topK: 10 });
```

### レコメンデーション

類似するアイテムを推薦します。

```typescript
// ユーザーの好みをベクトルで表現
const userVector = await embedUserPreferences(user);

// 類似するアイテムを検索
const recommendations = await vectorDB.search(userVector, { topK: 5 });
```

## AI開発における重要性

AI開発において、ベクトルDBは以下の用途で重要です：

- **RAG（Retrieval-Augmented Generation）**: 関連文書を検索してプロンプトに追加
- **セマンティック検索**: 意味に基づく検索
- **類似コンテンツの推薦**: ユーザーに類似コンテンツを推薦

## まとめ

- **ベクトルDB**は高次元のベクトルデータを効率的に格納・検索
- ベクトル検索により、類似度に基づく検索が可能
- AI開発において、RAGやセマンティック検索に重要
